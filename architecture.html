nacos-ops-dashboard 架构

本地 Nacos 实例管理面板 — 单文件 HTML + 自带 nginx,跑在 18080

darwin arm64 nginx 1.31.2 bash 3.2 + BSD nc

静态资源路径

浏览器请求 /dashboard/,nginx alias 直接 serve 本地 index.html。零后端依赖,纯静态。

动态配置路径

浏览器 JS fetch /__admin__/api/nacos-url,nginx 反代到 127.0.0.1:18081 的 reload-server.sh。

反向代理路径

浏览器请求 /nacos/*,nginx proxy_pass 转发到 127.0.0.1:8848 的 Nacos API,响应原路返回。

self-loop:配置回写

reload-server.sh 改 nginx-local.conf 的 proxy_pass 行 + nginx -s reload,下次 /nacos/ 请求走新地址。

用户操作 → 地址生效的完整流程

1
用户在浏览器粘完整地址 http://172.18.66.210:8848/nacos/#/serviceManagement?namespace=tools-dev,点"应用"
2
前端 extractNacosBaseUrl() 用正则智能提取 host = 172.18.66.210:8848,规范化成 http://172.18.66.210:8848/nacos/
3
fetch POST /__admin__/api/nacos-url,nginx 反代到 reload-server.sh:18081
4
reload-server.sh:① 改 nginx-local.conf 的 proxy_pass 行 → ② 重新渲染 .nginx.rendered.conf → ③ nginx -s reload
5
nginx master 收到信号,workers 平滑切换到新配置,后续 /nacos/* 全部走 172.18.66.210:8848
6
前端弹窗关闭,工具栏的"连接"按钮直接拿这个新地址去拉实例列表
